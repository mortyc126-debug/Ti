# БондАналитик: сбор раскрытий с e-disclosure.ru

Chrome/Edge/Brave расширение, которое собирает таблицу существенных
фактов с e-disclosure.ru и копирует структурированный JSON в буфер
обмена. Потом этот JSON импортируется в приложение БондАналитик через
кнопку «📢 факты» → textarea → «Импортировать».

## Установка (5 минут, только один раз)

### Chrome / Edge / Brave

1. **Скачать папку `extension/`** из репозитория:
   - Открой https://github.com/mortyc126-debug/ti в Chrome
   - Нажми **«Code»** → **«Download ZIP»**
   - Распакуй архив. Найди папку `extension/` внутри распакованного.
   - Альтернативно: `git clone https://github.com/mortyc126-debug/ti.git`, внутри будет `extension/`.
2. **Открыть страницу расширений:**
   - В адресной строке набери `chrome://extensions/` (или `edge://extensions/`)
3. **Включить режим разработчика:**
   - В правом верхнем углу переключатель «**Режим разработчика**» → включить.
4. **Загрузить распакованное расширение:**
   - Появится кнопка «**Загрузить распакованное**» (Load unpacked).
   - В диалоге выбрать папку `extension/` (именно её, не архив, не родительскую).
5. **Готово** — в списке появится «БондАналитик: сбор раскрытий».

### Firefox

Firefox поддерживает расширения, но установка чуть другая — через
`about:debugging#/runtime/this-firefox` → «Load Temporary Add-on» →
выбрать `manifest.json`. Работает до перезапуска браузера. Для
постоянной установки в Firefox нужна подпись в Mozilla Add-ons store
(или developer mode в Firefox Developer Edition / Nightly).

Проще через Chrome.

## Использование

1. Открой e-disclosure.ru в браузере (напр. через кнопку «📢 факты» в
   БондАналитике).
2. Найди компанию через поиск, кликни по ней → открыться карточка.
3. На странице с таблицей раскрытий (URL `portal/files.aspx?id=...`
   или `portal/company.aspx?id=...`) появится синяя кнопка в правом
   верхнем углу: **«📋 Собрать раскрытия → БондАналитик»**.
4. Клик → расширение собирает таблицу → JSON в буфере.
5. Вернись в БондАналитик → кнопка «📢 факты» на том же эмитенте →
   вставь JSON в поле textarea → «Импортировать».

Данные остаются в твоём reportsDB. В шапке эмитента появится
бейдж «⚠ N раскрытия» если есть критичные события за 90 дней.

## Что собирает

Для каждой строки таблицы:
- `date` — дата публикации (DD.MM.YYYY)
- `title` — заголовок существенного факта
- `url` — ссылка на полный текст (PDF / HTML / RTF / DOC)

Плюс общие метаданные:
- `source: "e-disclosure"` — маркер формата для импорта
- `edId` — внутренний ID e-disclosure (из URL `?id=...`)
- `companyName` — из `<h1>` или `<title>` страницы
- `inn` — если найден в тексте возле слова «ИНН»
- `capturedAt` — timestamp сбора

## Что не делает

- Не отправляет данные куда-либо в сеть (работает полностью локально)
- Не сохраняет ничего в расширении (только в буфер обмена)
- Не категоризирует события — это делает приложение при импорте
- Не работает вне e-disclosure.ru

## Безопасность

Расширение запрашивает минимум прав:
- `activeTab` — только на активной вкладке, только когда нажата кнопка
- `clipboardWrite` — запись в буфер
- `host_permissions` — только `*://*.e-disclosure.ru/*`, другие сайты
  не трогает

Полный код — `content.js` в этой папке. 110 строк, читается за 3 минуты.

## Обновление

Если захочешь обновить расширение (выйдет новая версия):
1. Перетяни из репозитория новую папку `extension/` поверх старой
2. На `chrome://extensions/` нажми иконку обновления рядом с
   расширением (↻ «Обновить»)

## Удаление

`chrome://extensions/` → кнопка «Удалить» рядом с расширением.
