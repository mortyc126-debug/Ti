## 6. OI — диагностика данных (аудит отложен)


Структура `oi_daily.json`: ключ = фьючерсный секид (NGU6), строка
`{tradedate, price, yur_long, yur_short, fiz_long, fiz_short, long, short, src_ticker}`.
Строк с price всего **387** (по 3–7 на инструмент) — только живой поллинг за ~неделю.

контрактов находит только текущий фронт (SBERF), не историческую цепочку.

**Вердикт:** OI-истории нет (~неделя живого поллинга, тонко). AlgoPack исторический
FutOI вглубь не отдаётся через текущий backfill. Аудит `oi_signal_dataset.py`
готов и проверен на синтетике, но ждёт накопления живого OI (~3–4 недели) либо
починки backfill (полная цепочка контрактов + цена из MOEX ISS).
