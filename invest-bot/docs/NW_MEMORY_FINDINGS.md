Живой рабочий журнал по новому методу (концепция
`kontseptsiya_temperatura_davlenie_pamyat_2.md`). Ведётся параллельно

Статус: **прототип валидирован walk-forward; заведён в бот как
`NWMemoryStrategy` (в фабрике), но в живом конфиге не активирован** —
`STRATEGY_OVERRIDE` пока на `AccelFadeStrategy`. Активация — вручную (см.

---

## Что это

Три оси мгновенного состояния бара (Layer 1 документа), уже считаются в
`tpcolor_dataset.py`:

- **T** (температура) = `volume × range / ATR_N` — интенсивность участия,
  direction-agnostic.
- **P** (давление) = Efficiency Ratio Кауфмана на окне N — направленность
  (0 = шум, 1 = чистый тренд).
- **color** = сглажённое ускорение цены `EMA(v[i]−v[i−N])`, `v=ROC(close,N)`.

Каждая ось нормируется каузальным rolling z-score (окно `w_norm≈500`, только
прошлое — §11.1). Целевая переменная — `fwd_ret_k = (close[i+k]−close[i])/ATR_N[i]`,
`target = sign(fwd_ret_k)`, горизонт `k=12` баров (5-мин → 60 мин вперёд).

**NW-память** (`nw_memory.py`, Layer 4 / §11): для текущего бара берёт
исторических соседей в пространстве (T̂,P̂,color̂), взвешивает гауссовым
ядром `exp(−d²/2h²)`, `p_hold = Σw·1{target>0}/Σw`. Ключевые фиксы из §11
реализованы:

- §11.3 лукахед-фильтр: сосед `j` используется только если `j+k ≤ i`.
- §11.2 dir_match: жёсткий множитель {0,1} по совпадению `sign(color)`.
- §5.4 честный «no precedent»: если density < порога → p_hold = NaN.

## Что подтверждено на данных (5-мин кэш, годы истории)

### §8 — оси независимы
Пул 368 тикеров, медианные Пирсон-корреляции:
`T̂↔P̂ = +0.08`, `T̂↔color̂ = 0.00`, `P̂↔color̂ = −0.03`. `|corr|>0.7`:
0/356 у первых двух. **Это три оси, а не три названия одной.**

### §8.3 — эффект локализован в квадранте, не глобален
Глобальная NW-память по всему пространству даёт edge ≈ 0 (сигнал
размывается по baseline). Локализация в квадрант **низкая T + высокая P**
(§8.3) поднимает edge в разы. Но:

- Пороги квадранта **обязаны быть per-ticker percentile**, не глобальный
  z-score. Глобальный `-0.5/+0.5` попадал «в никуда» у CBOM (0 точек),
  переворачивал знак у IRAO. Локальный `p5/p90` чинит.

### Walk-forward (честный out-of-sample) — гипотеза устояла
`--train-frac 0.6`: память + калибровка + percentile-пороги обучены на
первых 60% истории, метрики на последних 40% (модель не видела).

| тикер | holdout n | edge_raw (ATR) | Sharpe | win_raw |
|---|---|---|---|---|
| IRAO | 102 | **+1.64** | +0.47 | **68%** |
| CBOM | 156 | **+0.85** | +0.22 | 57% |
| PLZL | 243 | **+0.74** | +0.22 | 60% |
| SBER | 78 | −0.44 | −0.11 | 39% |

Batch walk-forward по 29 глубоким тикерам: **20/29 (69%) edge_raw > 0**
на holdout, медиана +0.26 ATR, 7/29 сильных (edge>0.3 И win>55%). ~69%
совпало с in-sample → **overfit'а нет**.

### Ликвидность: edge затухает с ростом ликвидности (монотонно)

Чистый прогон `lowT_highP @ p90`, 29 глубоких тикеров, трети по ликвидности:

| треть по ликв. | медиана edge_raw | edge>0 |
|---|---|---|
| низкая | **+0.629 ATR** | 78% (7/9) |
| средняя | +0.331 | 67% (6/9) |
| высокая | +0.141 | 55% (6/11) |

**Метод тем сильнее, чем менее ликвиден тикер.** Это НЕ знак-флип: верхняя
треть всё ещё в плюсе (+0.14/55%), так что «SBER-класс инвертирован» — про
отдельные имена-выбросы (SBER лично −0.44), не про весь ликвидный слой.
`Spearman(edge, log ликв)=−0.13` слабый (per-ticker edge шумный), но трети
показывают явный градиент. Волатильности НЕ верить: `Pearson +0.51` надут
выбросами, `Spearman` всего +0.11.

**⚠️ Конфаунд «длина истории».** Неликвидные тикеры обычно и короче по
истории → edge на них считается на меньшей выборке → медианы экстремальнее.
Разделить «ликвидность» и «длина истории» на текущем кэше нельзя, пока они
скоррелированы. В `_liq_vol_report` добавлены диагностики: `edge↔история`,
`ликвидность↔история` и трети ещё и по истории (с медианой длины в каждой
трети). Правила чтения:
- если `edge↔история` слабая, а `edge↔ликв` держится → градиент реально про
  ликвидность;
- если обе сильные И `ликв↔история` высокая → факторы неразделимы, нужна
  **докачка истории** коротким тикерам, потом перепрогон;
- параллельная страховка (без докачки): поднять `--batch-min-points` (напр.
  300) — если градиент держится только на хорошо-выборочных именах, это не
  артефакт короткой истории.

### Проверка других квадрантов (batch по всем 5 зонам, train-frac 0.6)

Прогнали `--zone {lowT_highP|highT_highP|lowT_lowP|highT_lowP}` по всему кэшу.
**Вывода два — методический и содержательный.**

**Методический (важный баг, уже починен).** Короткие фьючерсы (BNZ5, SXU6,
DXH6, S0Z5, ANH6, BSH6/BSM6, 95M6/95N6, ETU5, MEU5, SNM6, SCZ5, FSH6, CEZ5,
MTM6, RDM6, TYU6 …) показывали ОДИНАКОВЫЕ `n_quad`/`edge` во **всех** зонах
и абсурдные ±4…13 ATR. Причина: у них train-часть (первые 60%) короче окна
нормировки `w_norm≈500`, T̂/P̂ там сплошь NaN → `_zone_conditions` выкидывает
условие → маска зоны пустая → фильтр не применяется, edge считается по всем
барам. Такие строки загрязняли медианы `--zone`-прогонов. Фикс: если порог
зоны не задаётся для всех осей — тикер пропускается (`zone_undefined`).
**Все `--zone`-прогоны до фикса надо перегнать — их сводки не читаемы.**

**Содержательный (по ликвидам, вручную отбросив фьючерсы-артефакты):**

- **Порог P важнее, чем кажется: p90, не p95.** Первый рабочий прогон был
  `--t-pctl 5 --p-pctl 90` (T̂<p5 & P̂>p90) → +0.26 ATR, 69%. А `--zone
  lowT_highP` без ручных порогов берёт дефолт hi=p95 (T̂<p5 & P̂>**p95**) —
  и edge разваливается. То есть верхние 10% P работают, верхние 5% —
  переужимают в шум/exhaustion. **Рабочая зона = lowT_highP именно с p90.**
- **Второй универсальной зоны нет.** `highT_highP` частый (n_quad тысячи),
  но медиана edge на ликвидах ≈0; сильные лишь точечно (FIVE +1.36/58%,
  GDU6 +1.94/60%, SMLT +0.43/60%, VTBR +0.46/56%) — это отбор отдельных
  тикеров, не класс. `highT_lowP` — шум (медиана ~0). `lowT_lowP` — медиана
  на ликвидах отрицательная (GMKN −0.35, ALRS −0.83, NLMK −0.68, NVTK −0.60,
  RTKM −0.70), т.е. слабый ANTI-угол, но не чистый (SBER/GAZP там +).
- **Итог:** `lowT_highP @ p90` остаётся единственной робастной зоной.
  Наращивать частоту сигналов за счёт других квадрантов не вышло.

Перегнать после фикса (пример):
```bash
# рабочая зона — всегда с явным p90
python nw_memory.py --batch --tickers ALL --quadrant-only --zone lowT_highP \
    --t-pctl 5 --p-pctl 90 --train-frac 0.6 --batch-min-points 150 --out nw_lowThighP.csv
# остальные зоны — чтобы сводки очистились от фьючерсов-артефактов
python nw_memory.py --batch --tickers ALL --quadrant-only --zone lowT_lowP \
    --train-frac 0.6 --batch-min-points 150 --out nw_lowTlowP.csv
```

## Важные уроки (не потерять)

1. **Калибровка (изотоническая) НЕ помогает out-of-sample.** In-sample
   давала красивый +11% Brier, на holdout ≈ ноль. Настоящий сигнал — в
   **raw p_hold**. Для бота берём raw, калибровку выбрасываем.
2. **SBER — отдельный класс.** raw edge перевёрнут (−0.44), а не
   отсутствует. Крупнейшие ликвиды / news-driven (AFLT, FLOT, AFKS) —
   тоже «SBER-класс», инвертированная микроструктура. Их либо исключать,
   либо инвертировать.
3. **Сигнал редкий by design.** Квадрант p5×p90 = ~0.5% баров (~150-250
   событий/год на тикер). Регулируется порогами и/или другими зонами.
4. **Свечные fade-паттерны — то же явление другой стороной.** На 5-мин
   «длинная свеча в направлении X» (Marubozu/BeltHold/LongLine) работает
   как anti-сигнал (win 38-41% на 800k-1.3M наблюдений). Уже добавлено в
   бот как метод `TALIB_ANTISIGNAL`.

## Инструменты (готовы, в `invest-bot/`)

- `tpcolor_dataset.py` — считает T/P/color/fwd_ret по кэшу. single / ALL,
  per-ticker percentile, корреляции §8, quadrant-check §8.3, sweep.
- `nw_memory.py` — NW-память. single / `--batch`. Ключевые флаги:
  `--quadrant-only --zone {lowT_highP|highT_highP|lowT_lowP|highT_lowP|…}`,
  `--t-pctl/--p-pctl`, `--train-frac` (walk-forward), `--calibrate`
  (изотоническая, для диагностики), `--confidence`. Batch пишет по тикеру
  `liq_mln` (медианный барный оборот close·volume) и `vol_pct` (медианный
  относит. диапазон), а в конце — корреляции `edge_raw ↔ ликвидность/
  волатильность` (Pearson+Spearman) и медиану edge по третям ликвидности
  (ловит немонотонность — «топ-ликвиды инвертированы», SBER-класс).
- `candle_patterns.py` — 61 свечной паттерн TA-Lib по кэшу.
