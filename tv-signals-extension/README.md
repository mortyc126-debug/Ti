# Сигнальные модели на графике (Тинькофф веб-терминал / TradingView)

Считает «Сигнальные модели» (Z-score, NW-память, Order Block, FVG, Accel-fade,
VSA, Hawkes, Cascade и др.) на свечах **самого графика** и рисует их сигналы
**нативными фигурами TradingView** (стрелки, привязанные к цене/времени). Рядом —
точность каждого метода по текущему тикеру.

Данные берутся из графика (`exportData()`) — **токен не нужен**.

## Как это работает (технически)

Терминал Тинькофф рисует график встроенным TradingView Charting Library в
same-origin iframe; его API доступен как `iframe.contentWindow.tradingViewApi`.
Расширение (content-script в MAIN-мире верхней страницы):
1. дожидается готовности графика,
2. `activeChart().exportData()` → свечи (парсинг по `schema`, колонки динамические),
3. `SignalsCore.computeAll()` → сигнал + точность (win% за 12 баров) по каждому методу,
4. клик по методу → `activeChart().createShape({time,price},{shape:'arrow_up/down'})`
   рисует его сигналы в видимом окне (до 60 последних).

## Установка

1. `chrome://extensions` → «Режим разработчика».
2. «Загрузить распакованное» → папка `tv-signals-extension/`.
3. Открыть терминал `tbank.ru/terminal` → справа панель «◆ Сигнальные модели».

## Панель

- Строка на метод: цветная метка · имя · текущий сигнал (▲buy/▼sell/—) · **точность** (win% по этому тикеру) · n сделок.
- **Клик по строке** — включить/выключить рисование сигналов метода на графике.
- **⟳** — пересчитать. **–** — свернуть. Заголовок — перетаскивание.
- Пересчёт автоматически при смене тикера (опрос ~2.5 c).

## Ограничения v1

- **Объём:** если в `exportData` объёма нет (зависит от того, включён ли индикатор
  объёма на графике), объёмные методы (VSA) деградируют. Планово — брать объём
  из `datafeed` конфига.
- Точность считается на видимой истории графика (сколько прогружено — столько и
  баров, обычно тысячи). Это оценка, не гарантия.
- Рисуются последние ~60 сигналов в видимом окне (чтобы не засорять/не тормозить).

## Проверено

Headless (Chromium) против мок-`tradingViewApi` той же формы, что подтверждена в
реальном терминале (`exportData`, `createShape`, `symbol`, `getVisibleRange`):
парсинг schema → 700 свечей, панель из 12 методов с точностью, клик вызывает
`createShape` (60 фигур), 0 ошибок. Реальная отрисовка/цвета — при запуске на
терминале (если стрелки не того цвета — правится ключ `overrides`).
